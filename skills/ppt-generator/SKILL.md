# PPT Generator

발표 주제, 대상 청중, 발표 시간을 입력받아 전문적인 .pptx 파일을 생성한다.

## When to Use

- 사용자가 PPT, 발표자료, 프레젠테이션, 슬라이드를 만들어달라고 할 때
- 발표 준비를 도와달라고 할 때
- 기존 내용을 PPT로 정리해달라고 할 때
- 회의, 세미나, 수업, 보고 등 어떤 목적이든 슬라이드가 필요할 때

## Input

사용자에게 다음 정보를 확인한다 (없으면 물어본다):

| 항목 | 필수 | 기본값 |
|------|------|--------|
| 발표 주제 | O | - |
| 대상 청중 | X | 일반 직장인 |
| 발표 시간 | X | 10분 |
| 언어 | X | 한국어 |
| 톤 | X | 전문적이되 이해하기 쉬운 |

## Workflow

### 1단계: 슬라이드 구성 설계

발표 시간에 따라 슬라이드 수를 결정한다:

| 발표 시간 | 슬라이드 수 | 설명 |
|-----------|------------|------|
| 5분 | 5~7장 | 핵심만 간결하게 |
| 10분 | 8~12장 | 표준 발표 |
| 15분 | 12~18장 | 상세 발표 |
| 20분 | 16~22장 | 심화 발표 |
| 30분+ | 22~30장 | 세미나/강의 |

슬라이드는 다음 구조를 따른다:

```
1. 표지 (제목 + 발표자 + 날짜)
2. 목차 / 아젠다
3~N-2. 본문 슬라이드들
N-1. 요약 / 핵심 정리
N. 마무리 (Q&A 또는 감사 인사)
```

### 2단계: 각 슬라이드 상세 설계

각 슬라이드에 다음 3가지를 반드시 포함한다:

1. **제목**: 슬라이드의 주제를 한 줄로 (15자 이내 권장)
2. **핵심 메시지**: 이 슬라이드에서 전달할 내용 (불릿 3~5개 또는 짧은 문단)
3. **시각 자료 지시사항**: 어떤 차트, 다이어그램, 이미지가 적합한지 설명
   - 예: "막대 그래프로 연도별 매출 비교", "프로세스 플로우 다이어그램", "아이콘 4개로 핵심 가치 표현"

### 3단계: 슬라이드 구성안을 사용자에게 보여주기

.pptx를 생성하기 전에, 전체 슬라이드 구성안을 텍스트로 먼저 보여준다.
사용자가 확인하고 수정 요청이 없으면 다음 단계로 진행한다.

### 4단계: .pptx 파일 생성

`scripts/generate_pptx.js` 스크립트(PptxGenJS 기반)를 사용하여 .pptx 파일을 생성한다.

슬라이드 데이터를 JSON으로 만들어 스크립트에 전달한다:

```bash
node <skill-path>/scripts/generate_pptx.js \
  --input slides.json \
  --output "<제목>.pptx" \
  --theme <테마>
```

JSON 형식:
```json
{
  "title": "발표 제목",
  "presenter": "발표자명",
  "date": "2025-01-15",
  "theme": "professional",
  "slides": [
    {
      "type": "title",
      "title": "발표 제목",
      "subtitle": "부제목 또는 발표자"
    },
    {
      "type": "agenda",
      "title": "목차",
      "items": ["항목1", "항목2", "항목3"]
    },
    {
      "type": "content",
      "title": "슬라이드 제목",
      "bullets": ["불릿1", "불릿2", "불릿3"],
      "visual_note": "시각 자료 지시사항"
    },
    {
      "type": "two_column",
      "title": "비교 슬라이드",
      "left_title": "왼쪽 제목",
      "left_bullets": ["항목1", "항목2"],
      "right_title": "오른쪽 제목",
      "right_bullets": ["항목1", "항목2"]
    },
    {
      "type": "quote",
      "title": "인용",
      "quote": "인용문",
      "attribution": "출처"
    },
    {
      "type": "summary",
      "title": "핵심 정리",
      "bullets": ["요약1", "요약2", "요약3"]
    },
    {
      "type": "closing",
      "title": "감사합니다",
      "subtitle": "Q&A"
    }
  ]
}
```

사용 가능한 테마:
- `professional`: 네이비 + 화이트, 깔끔한 비즈니스 스타일 (기본값)
- `modern`: 다크 그레이 + 포인트 컬러, 세련된 느낌
- `warm`: 따뜻한 톤, 교육/세미나용
- `minimal`: 흰 배경 + 블랙 텍스트, 미니멀 스타일
- `bold`: 강렬한 컬러, 임팩트 있는 발표용

### 5단계: 검증

.pptx 생성 후 다음 체크리스트로 검증한다:

#### 구조 검증
- [ ] 표지 슬라이드에 제목, 발표자, 날짜가 있는가
- [ ] 목차 슬라이드가 본문 내용과 일치하는가
- [ ] 마지막 슬라이드가 Q&A 또는 마무리인가
- [ ] 슬라이드 수가 발표 시간에 적합한가

#### 내용 검증
- [ ] 각 슬라이드에 제목이 있는가
- [ ] 한 슬라이드에 불릿이 6개를 넘지 않는가
- [ ] 시각 자료 지시사항이 모든 본문 슬라이드에 포함되어 있는가
- [ ] 전문 용어가 대상 청중 수준에 맞는가

#### 흐름 검증
- [ ] 논리적 순서로 이야기가 전개되는가
- [ ] 서론-본론-결론 구조가 명확한가
- [ ] 슬라이드 간 자연스러운 전환이 가능한가

#### 디자인 검증
- [ ] 텍스트가 너무 많지 않은가 (슬라이드당 50단어 이내 권장)
- [ ] 일관된 디자인 테마가 적용되었는가
- [ ] 폰트 크기가 가독성이 좋은가 (제목 28pt+, 본문 18pt+)

### 6단계: 전달

생성된 파일을 사용자에게 안내한다:
- 파일 경로
- 총 슬라이드 수
- 적용된 테마
- 시각 자료 지시사항이 포함된 슬라이드 목록 (사용자가 직접 이미지/차트를 추가할 수 있도록)

## Content Guidelines

### 슬라이드 작성 원칙

- **1슬라이드 1메시지**: 하나의 슬라이드에서 전달하는 핵심은 하나
- **키워드 중심**: 장문의 문장 대신 핵심 키워드와 짧은 구문
- **불릿 3~5개**: 한 슬라이드의 불릿은 3~5개가 이상적
- **시각적 균형**: 텍스트와 빈 공간의 적절한 배분
- **청중 눈높이**: 전문 용어 수준을 대상 청중에 맞춤

### 시각 자료 지시사항 작성법

각 슬라이드의 `visual_note`에는 발표자가 나중에 추가할 시각 자료에 대한 구체적 안내를 적는다:

**좋은 예:**
- "2020~2024년 매출 추이를 보여주는 꺾은선 그래프"
- "4단계 프로세스를 화살표로 연결한 플로우 다이어그램"
- "시장 점유율을 보여주는 도넛 차트 (당사 35%, 경쟁사A 25%, 기타 40%)"

**나쁜 예:**
- "적절한 이미지 삽입" (너무 모호)
- "그래프" (어떤 종류인지 불분명)

## Examples

**Example 1:**
Input: "AI 트렌드에 대해 15분 발표할 건데, 대상은 비개발 임원진이야"
Output: 15장 슬라이드 구성 → .pptx 파일 생성
- 표지, 목차, AI 시장 현황, 주요 트렌드 3~4개, 우리 회사 적용 방안, 요약, Q&A
- 전문 용어는 최소화하고 비즈니스 임팩트 중심
- 테마: professional

**Example 2:**
Input: "우리 팀 프로젝트 결과 보고 PPT 만들어줘, 5분짜리"
Output: 6장 슬라이드 구성 → .pptx 파일 생성
- 표지, 프로젝트 개요, 주요 성과, 데이터/지표, 향후 계획, 마무리
- 간결하고 데이터 중심
- 테마: professional

**Example 3:**
Input: "초등학생 대상 환경보호 수업 PPT 30분"
Output: 25장 슬라이드 구성 → .pptx 파일 생성
- 흥미로운 질문으로 시작, 사진/일러스트 중심, 퀴즈 슬라이드 포함
- 쉬운 언어, 큰 글씨
- 테마: warm
